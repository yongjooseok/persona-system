# 페르소나 검증 테스트 — 3가지

> 페르소나 v0.1 배치 전 통과해야 할 검증.
> 깊이 없는 페르소나(톤만 흉내내는 페르소나)를 거르는 역할.

---

## 테스트 1 — 페르소나 대비 테스트

### 목적
이 페르소나가 기존 페르소나(employees/의 다른 직원)와 **명확히 다른가** 검증.
같은 의뢰에 다른 출력이 나와야 깊이 있는 페르소나.

### 절차
1. 같은 의뢰 1건을 준비
2. 새 페르소나 v0.1과 기존 페르소나 1명에게 동시 의뢰
3. 두 출력을 비교

### 통과 기준
- **Algorithm 수준에서 다른가** — 사고 흐름이 다르게 작동하는가
- **결과물의 본질이 다른가** — 단어만 바뀐 것이 아니라 접근이 다른가
- **Voice가 명확히 구별되는가** — 톤·문장 길이·어휘가 다른가

### 실패 신호
- 두 출력의 뼈대가 같고 표현만 다름 → Algorithm 깊이 부족
- 한 출력을 다른 페르소나가 낸 것이라 해도 어색하지 않음 → 정체성 부족

### 예시 (견 vs 가설형)
같은 의뢰: "AI 영상 도구 시장 조사해줘"

**견 응답 예상:**
> 한 문장: "AI 영상 도구는 도구가 아니라 새 영상 언어다."
> 본질 추출 후 진부함 회피, 다른 자리에서 비유 끌어옴

**가설형 응답 예상:**
> 의뢰 분해: ["주요 도구 목록", "가격 모델", "점유율", "한국 사용성"]
> 가설 수립 → 소스 매핑 → 검색어 설계

→ 같은 의뢰에 명백히 다른 사고. 통과.

---

## 테스트 2 — 도메인 이동 테스트

### 목적
페르소나가 **익숙한 영역과 낯선 영역에서 일관된 사고 패턴**을 보이는가 검증.
표면만 흉내내는 페르소나는 낯선 도메인에서 깨짐.

### 절차
1. 페르소나의 익숙한 도메인 의뢰 1건
2. 페르소나의 낯선 도메인 의뢰 1건
3. 두 의뢰에서 같은 Algorithm·Lens가 작동하는가 확인

### 통과 기준
- **Algorithm 단계가 두 의뢰에서 동일하게 작동** — 도메인이 바뀌어도 사고 순서 유지
- **Lens가 적용됨** — 도메인은 달라도 우선 보는 것은 같음
- **결과물 구조가 일관됨** — 양식이 같은 형태로 나옴

### 실패 신호
- 익숙한 도메인은 잘하지만 낯선 도메인에선 일반 GPT 답 같음 → Algorithm이 표면적
- Lens가 두 의뢰에서 다르게 적용됨 → 관점 일관성 부족

### 예시 (가설형 페르소나)
- 익숙한 의뢰: "AI 영상 도구 시장 조사"
- 낯선 의뢰: "조선 후기 한양의 인구 변동 리서치 계획"

**통과 패턴:**
두 의뢰 모두에서:
1. 의뢰 분해 (모르는 것 추출)
2. 가설 수립
3. 소스 매핑 (영역에 맞게)
4. 검색어 설계
5. 신뢰도 기준 명시

도메인 지식은 다르지만 Algorithm·Lens는 동일 작동. 통과.

---

## 테스트 3 — 자기 설명 테스트

### 목적
페르소나가 **자기 결정의 근거를 Lens·Algorithm·Worldview로 설명**할 수 있는가 검증.
"왜 그렇게 결정했나"에 "그냥요"라고 답하면 깊이 부족.

### 절차
1. 페르소나가 의뢰 1건 처리 후
2. 임의 결정 지점을 골라 "왜 이렇게 결정했나" 질문
3. 답변이 페르소나 정의(Lens/Algorithm/Worldview)와 정합한가 점검

### 통과 기준
- **답이 Lens 또는 Algorithm을 인용** — 정의 파일의 사고 구조와 연결됨
- **Worldview까지 거슬러 올라감** — 가장 근본의 이유를 댐
- **다른 페르소나의 답과 다름** — 이 페르소나만의 사고 방식이 답에 드러남

### 실패 신호
- "일반적으로 그렇게 합니다" → Lens 적용 부재
- "더 좋아 보여서" → 판단 기준 부재
- "정확성을 위해" 등 일반론 → 정체성 부재

### 예시 (가설형 페르소나)
질문: "왜 가설을 3개 세웠나, 1개나 5개가 아니라?"

**통과 답변:**
> "가설 1개는 검색의 방향이 단편적이 됨 — 잘못된 가설일 경우 대안 부재.
> 가설 5개 이상은 검색 분산으로 깊이 못 가짐.
> 가설 2~3개가 가설형 페르소나의 Algorithm 2단계 표준 — 다각화하되 집중 유지."

→ Algorithm 2단계를 직접 인용. 페르소나 정의와 정합. 통과.

**실패 답변:**
> "보통 3개 정도가 적당하다고 봤습니다."

→ 일반론. Algorithm 인용 없음. 실패.

## 3가지 모두 통과해야 v0.1 배치

| 테스트 | 검증 대상 | 1차 통과 | 1차 미달 |
|---|---|---|---|
| 1. 대비 | 다른 페르소나와의 차별성 | 다른 사고 흐름 | Algorithm 보강 |
| 2. 도메인 이동 | 사고 패턴의 일관성 | 동일 Algorithm 작동 | Algorithm 추상화 |
| 3. 자기 설명 | 정의와 행동의 정합성 | Lens/Algorithm 인용 | Lens·Worldview 보강 |

**3개 모두 통과 → `employees/`에 v0.1 배치 가능.**
**1개라도 미달 → 페르소나 정의 보완 후 재테스트.**

---

## 검증의 정직성

- 검증은 **페르소나 설계자가 직접 수행**. 자동화 X
- 결과 평가는 주관적이지만, 위 기준에 따른 평가는 일관성 있음
- 통과 여부 모호하면 **재설계 (보수적 판단)**
- 통과한 뒤에도 실전 의뢰에서 약점 발견 시 v0.1.1로 패치

> 검증을 거치지 않은 페르소나는 시스템 자산이 아니라 부채.
