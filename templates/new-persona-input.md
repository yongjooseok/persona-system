# 신규 페르소나 입력 양식

> 의뢰인이 새 페르소나를 요청할 때 채우는 양식.
> 필수 3항목(1·2·3)만 채우면 시작 가능. 나머지는 페르소나 설계자가 보완·추천 가능.

---

## 필수 항목

### 1. 용도
이 페르소나가 무엇에 쓰일 것인가 (한 줄)

> 예: "Naver 광고 카피 작성"
> 예: "정부지원사업 제안서 검토"
> 예: "리서치 계획 수립 (웹 + NotebookLM)"

### 2. 핵심 문제
이 페르소나가 없으면 풀 수 없는 일은 무엇인가
- 현재 상황
- 현재의 불편·한계

> 예: "Creamhaus 신제품마다 Naver 키워드 광고 카피 30개씩 만드는데,
>      매번 비슷한 톤으로 빠지고 차별화가 안 됨"

### 3. 산출물 형태
이 페르소나가 무엇을 만들어내야 하는가
- 형식 (텍스트/표/리스트/구조화된 문서)
- 길이·개수
- 채널·매체 (필요 시)

> 예: "키워드 1개당 15자 이내 카피 5안, 표 형태"
> 예: "리서치 계획서 1건 (분해된 모르는 것 + 가설 + 검색어 + 평가 기준)"

---

## 선택 항목 (있으면 페르소나 깊이 향상)

### 4. 톤·관점
어떤 어조·관점이 필요한가
- 명시하지 않으면 페르소나 설계자가 용도에 맞춰 추정·제안

> 예: "전문적이지만 거리감 없는 톤"
> 예: "행동을 유발하는 강한 어조"
> 예: "데이터 우선 분석가 톤"
> "잘 모르겠음 → 추천 받음" 도 OK

### 5. 참조 인물·스타일
원형 차용 가능한 인물 또는 학파
- (가) 실존 인물 — 이름 + 차용하고 싶은 부분
- (나) 방법론 학파 — 학파명 또는 분야
- (다) 잘 모름 → 페르소나 설계자가 추천

> 예: "정철 카피라이터 풍"
> 예: "OSINT(공개정보분석) 학파"
> 예: "잘 모름, 추천"

### 6. 결정 권한 경계
이 페르소나가 결정할 수 있는 것 / 의뢰인이 결정할 것 구분
- 명시하지 않으면 기본 원칙 적용 ("최종 결정·창작 권위는 의뢰인")

> 예: "카피 후보 생성은 전권, 최종 카피 선택은 내가"
> 예: "리서치 계획 수립 전 과정 전권, 실제 검색·실행 여부는 내가 결정"

### 7. 도구
이 페르소나가 사용할 도구
- 프로젝트 내부 스킬 (skills/ 디렉토리)
- 외부 도구 (NotebookLM, MCP, 특정 API 등)
- 내장 전문 지식 (페르소나 정의에 흡수)
- 명시하지 않으면 페르소나 설계자가 적합한 것 추천

> 예: "NotebookLM, MCP(Exa·Firecrawl), 웹 리서치 전문 지식 내장"
> 예: "@skills/content-os, @skills/idea-brainstorm"

---

## 입력 후 진행

페르소나 설계자에게 위 양식 전달.
설계자가 부족한 항목을 점검 후:
- 필수 누락 → 의뢰인에게 보완 요청
- 선택 누락 → 추정·추천 후 의뢰인 컨펌

이후 `CREATING-NEW-PERSONA.md`의 5단계 절차 진행.
