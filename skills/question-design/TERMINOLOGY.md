# 질문 전문가 용어집

## 용도
- 질문 문제 진단 시 참조
- 고급 질문 기법 적용
- 질문이 잘못됐을 때 원인 파악

---

## 1. 질문 위치 용어

### 메타질문 (Meta-question)
- **정의**: 질문 자체를 대상으로 삼는 질문
- **사용 시점**: 질문해도 답이 이상할 때
- **예시**:
  - "내가 지금 잘못 묻고 있는 건 뭘까?"
  - "이 질문의 전제는 무엇인가?"
  - "이 질문을 버린다면, 대신 어떤 질문을 해야 하지?"

### 프레이밍 (Framing)
- **정의**: 문제의 해석 틀
- **핵심**: 프레임이 바뀌면 답이 완전히 달라짐
- **예시**:
  - "이건 기술 문제인가, 결정 문제인가?"
  - "이건 실력 문제인가, 구조 문제인가?"

### 전제 (Presupposition)
- **정의**: 질문에 숨어있는 가정
- **위험**: 틀린 전제 = 틀린 답
- **점검 질문**:
  - "이 질문은 무엇을 당연하게 가정하고 있나?"
  - "이 전제가 틀렸다면?"

---

## 2. 질문 기능 용어

### 정보 질문 (Exploratory)
- 목적: 모름 → 앎
- Level 1 해당

### 정렬 질문 (Alignment)
- 목적: 선택지 비교/기준 정리
- Level 2 해당

### 구조 질문 (Structural)
- 목적: 실행 가능한 형태로 분해
- Level 3 해당

### 위임 질문 (Delegation)
- 목적: 이해 제거, 결과 확보
- Level 4 해당

### 종료 질문 (Closing)
- **정의**: 더 묻지 않기 위한 질문
- **핵심**: 가장 고급 질문
- **예시**:
  - "이걸 다시 고민하지 않으려면 무엇을 정해야 하지?"
  - "이 질문 이후 더 물을 게 있나?"

---

## 3. 질문 깊이 용어

### 표면 질문 (Surface)
- 증상에 묻는다
- 예: "왜 안 되지?"

### 원인 질문 (Causal)
- 구조적 이유를 묻는다
- 예: "이게 반복해서 막히는 구조적 이유는?"

### 전제 질문 (Assumption)
- 암묵적 가정을 드러낸다
- 예: "이게 꼭 필요하다는 가정은 어디서 왔지?"

### 반전 질문 (Inversion)
- **정의**: 문제를 뒤집는다
- **효과**: 창의성 폭발 지점
- **예시**:
  - "이걸 실패하게 하려면 뭘 하면 될까?"
  - "이걸 안 해도 되는 방법은?"
  - "반대로 생각하면?"

---

## 4. 프롬프트 전문 용어

### 의도 명시 (Intent Declaration)
- "무엇을 알고 싶은지"가 아니라 "무엇을 바꾸고 싶은지"
- 예: "결정 범위를 줄이기 위해 묻는다"

### 제약 조건 (Constraints)
- AI의 창의성을 열거나 닫는 스위치
- 예: 시간=0, 돈=무제한, 기술 이해=불필요

### 역할 부여 (Role Assignment)
- AI의 사고 스타일 규정
- 예: "비판자 입장에서", "초고수 전문가라면"

### 출력 범위 제한 (Output Bounding)
- 답의 폭 관리
- 예: "3가지만", "행동 기준으로만"

### 질문 재작성 요청 (Question Rewriting)
- **정의**: 가장 고급 프롬프트 기법
- **사용법**: "이 질문보다 더 본질적인 질문으로 바꿔줘"

---

## 5. 질문 품질 지표

### 질문 밀도 (Question Density)
- **정의**: 질문 개수 대비 의사결정 정보량
- **원칙**: 고수일수록 질문 수↓, 밀도↑
- **자가 점검**: "이 질문 하나로 얼마나 많은 걸 알 수 있나?"

### 질문 잔존성 (Question Residue)
- **정의**: 답을 들어도 계속 남는 질문
- **신호**: ❌ 좋은 신호 아님
- **원인**: 질문이 잘못 설계됨
- **해결**: 메타질문으로 점검

### 질문 안전성 (Question Safety)
- **정의**: 질문이 너무 안전하면 답은 뻔하다
- **원칙**: 창의적 답변은 불편한 질문에서 나온다
- **예시**: "이게 틀렸을 가능성은?" vs "이게 맞지?"

---

## 6. 실전 도구 문장

### 질문 점검용
```
"이 질문의 전제부터 공격해줘"
"내 질문을 더 위험하게 만들어줘"
"이 질문이 막고 있는 가능성은 뭐야?"
"이 질문을 버린다면, 대신 어떤 질문을 해야 하지?"
```

### 답변 검증용
```
"이 답변이 틀렸다면 어디가 틀렸을까?"
"반대 의견이 있다면 뭘까?"
"이걸 믿으면 안 되는 경우는?"
```

### 확장용
```
"내가 놓치고 있는 게 뭐야?"
"이 문제를 완전히 다르게 접근한다면?"
"10배 나은 결과를 내려면 뭘 바꿔야 해?"
```

## 7. 문제 유형별 진단

| 문제 | 의심 용어 | 점검 질문 |
|------|-----------|-----------|
| 답이 뻔함 | 질문 안전성 | "더 불편한 질문은?" |
| 답 들어도 찜찜 | 질문 잔존성 | "뭐가 남아있지?" |
| 질문이 많아짐 | 질문 밀도 | "하나로 합칠 수 있나?" |
| 답이 내 상황과 안 맞음 | 프레이밍 | "틀이 맞나?" |
| 계속 빙글빙글 | 전제 오류 | "전제가 틀렸나?" |
| 진전이 없음 | 메타질문 필요 | "질문 자체가 맞나?" |
