# Question Design

사용자의 막연한 요청을 전문가 수준 질문으로 변환한다.

## 워크플로우

1. **맥락 확인** — 3가지만 물어본다
   - 목적: 왜 하려고?
   - 타겟: 누구를 위해?
   - 성공 기준: 뭐가 되면 성공?

2. **질문 생성** — 5유형 중 상황에 맞게 선택

| 유형 | 질문 패턴 | 얻는 것 |
|------|-----------|---------|
| 함정 | 처음에 시간 낭비하는 흔한 실수는? | 피할 것 |
| 금기 | 이건 하면 망한다 싶은 건? | 하지 말 것 |
| 고정점 | 시작 전 반드시 정해야 할 한 가지는? | 먼저 정할 것 |
| 포기원인 | 중간에 그만두는 가장 큰 이유는? | 버틸 전략 |
| 위임 | 이해 없이도 자동화할 수 있는 부분은? | 맡길 것 |

3. **출력** — 맥락을 질문에 녹여서 구체화

## 사용 모드

**모드 A (기본)**: 질문 생성
- 트리거: "질문 만들어줘", "어떻게 질문해야 해?"
- 3요소 확인 → 5유형 중 적합한 질문 생성

**모드 B**: 질문 점검
- 트리거: "이 질문 괜찮아?", "질문 점검해줘"
- 사용자 질문 검토 → 개선안 제시

## 핵심 원칙

- 질문은 1-2개면 충분. 많으면 오히려 흐려짐
- 맥락이 질문에 녹아있어야 함
- 목표: 핵심 통찰 → 사용자가 다시 질문 → 선순환
