# 질문 레벨 4단계 시스템

## 용도
- 현재 질문의 레벨을 판단
- 레벨에 맞는 응답 제공
- 다음 레벨로 전환 유도

---

## 레벨 개요

| Level | 이름 | 목적 | 키워드 |
|-------|------|------|--------|
| L1 | 정보 질문 | 혼란 제거 | "이게 뭐야?" |
| L2 | 방법 질문 | 선택지 정렬 | "A vs B?" |
| L3 | 구조 질문 | 실행 설계 | "어떻게 나눠?" |
| L4 | 위임 질문 | 확장/수익화 | "알아서 해줘" |

**핵심 원칙**: L1-2는 통과 지점, L3-4가 목적지

---

## Level 1 — 정보 질문 (혼란 제거)

### 언제 사용?
- 원인을 모를 때
- 개념/용어가 헷갈릴 때
- 전체 그림이 안 보일 때

### 허용 질문
```
□ 이게 뭔가?
□ 가능한 선택지는 뭐야?
□ 지금 이 분야에서 쓰이는 개념은?
□ 이걸 뭐라고 부르지?
```

### 🚨 정지 조건
```
이 정보로 바로 결정하려 하고 있지 않은가?
→ 결정하려 한다면 L2로 이동해야 함
```

### ➡️ L2 전환 질문
```
"정보는 충분한가, 선택을 해야 하는가?"
```

---

## Level 2 — 방법 질문 (선택지 정렬)

### 언제 사용?
- 여러 경로 중 고르려 할 때
- 비용/시간/난이도 비교가 필요할 때
- "어떻게"가 궁금할 때

### 허용 질문
```
□ A와 B의 차이는?
□ 일반적인 순서는?
□ 초보자 기준 현실적인 방법은?
□ 각각의 장단점은?
□ 내 상황에선 뭐가 나아?
```

### 🚨 정지 조건
```
아직 "내 경우"를 확정하려 하고 있는가?
→ L2는 비교 단계, 확정은 L3에서
```

### ➡️ L3 전환 질문
```
"이 중 하나를 반복할 계획이 있는가?"
```

---

## Level 3 — 구조 질문 (실행 설계)

### 언제 반드시 올라와야 하나?
- 반복할 계획이 생겼다
- 자동화/AI/코드/외주를 고려 중이다
- 프로세스를 만들려 한다

### 핵심 질문
```
□ 사람이 해야 할 일 / AI가 할 일을 나눴는가?
□ 시작 전에 고정해야 할 단 하나는 무엇인가?
□ 이 과정을 코드 없이 문장으로 설명할 수 있는가?
□ 실패할 가능성이 가장 높은 지점은?
□ 이 구조를 반복하면 어떤 결과가 나오는가?
```

### 🚨 정지 조건
```
여기서 다시 공부 질문(L1)으로 내려가고 있지 않은가?
→ 개념이 불확실하면 명시적으로 "L1으로 내려감" 선언 후 이동
```

### ➡️ L4 전환 질문
```
"이해하지 않아도 되게 만들어도 되는가?"
```

---

## Level 4 — 위임 질문 (확장/수익화)

### 언제 사용?
- 결과가 이해보다 중요하다
- 반복/확장/복제를 원한다
- 시스템으로 만들려 한다

### 핵심 질문
```
□ 내가 이해하지 않아도 돌아가게 설계할 수 있는가?
□ 실패 가능성 높은 지점을 먼저 정의했는가?
□ 이 구조를 템플릿으로 복제할 수 있는가?
□ 누가 해도 같은 결과가 나오는가?
□ 이 시스템의 유지보수 비용은?
```

### 완료 조건
```
이 질문 이후, 더 이상 묻지 않아도 되는가?
→ "예"라면 종료
→ "아니오"라면 어떤 질문이 남았는지 확인
```

---

## 레벨 전환 스위치 (한눈에)

```
L1 → L2: "정보는 충분한가, 선택을 해야 하는가?"
L2 → L3: "이 중 하나를 반복할 계획이 있는가?"
L3 → L4: "이해하지 않아도 되게 만들어도 되는가?"
```

---

## 레벨 에러 진단

### 증상 → 원인 → 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 같은 질문 반복 | 레벨이 안 올라감 | 전환 스위치 질문 |
| 답 들어도 뭔가 찜찜 | 질문 레벨 ≠ 실제 필요 레벨 | 레벨 재판단 |
| 정보만 쌓이고 진전 없음 | L1에서 정체 | 강제로 L2 질문 |
| 비교만 하고 결정 못함 | L2에서 정체 | "하나 고른다면?" |
| 구조는 있는데 실행 안됨 | L3 설계 불완전 | 역할 분리 재점검 |

---

## AI 응답 시 출력 형식

```
[Level X - 레벨명]
현재 상황 요약 (1줄)
→ 답변 또는 제안
→ 다음 질문: "전환 스위치 질문" (Level X+1 전환 검토)
```

예시:
```
[Level 2 - 방법 질문]
React vs Vue 비교 중이에요.
→ 초보자 기준으로는 Vue가 진입장벽이 낮아요. React는 생태계가 넓어요.
→ 다음 질문: "이 중 하나를 반복 프로젝트에 쓸 계획인가요?" (Level 3 전환 검토)
```

---

## 레벨 역행 규칙

### 허용되는 경우
- 구조 설계 중 개념이 불확실해짐 → L1으로
- 위임 설계 중 선택지가 더 필요함 → L2로

### 반드시 해야 할 것
```
"[이유]로 Level X에서 Level Y로 내려갑니다"
```

예시:
```
"OAuth 구조 설계 중인데 토큰 갱신 개념이 불확실해서 L1으로 내려갑니다"
```
