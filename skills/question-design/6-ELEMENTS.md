# 첫 질문 6요소

## 용도
- 프로젝트 시작 시 가장 먼저 정리
- Phase 1 (자기 정리) 단계에서 사용
- AI에게 질문하기 전 필수 선행

---

## 6요소 정의 (순서대로)

| 순서 | 요소 | 질문 | 예시 |
|------|------|------|------|
| 1 | **목적** | 왜 하려고? | 수익화 / 학습 / 자동화 / 문제 해결 |
| 2 | **타겟** | 누구를 위해? | 나 / 고객 / 특정 집단 / 불특정 다수 |
| 3 | **현재 상태** | 지금 어디야? | 아이디어만 / 절반 진행 / 막힘 |
| 4 | **제약** | 조건이 뭐야? | 도구 / 시간 / 예산 / 실력 수준 |
| 5 | **결과물** | 뭐가 나오면 돼? | 웹페이지 / 스크립트 / 시스템 / 문서 |
| 6 | **성공 기준** | 언제 끝이야? | 배포 완료 / 작동 확인 / 매출 발생 |

**순서 설계 의도:**
- 목적/타겟 먼저 → 방향 고정
- 현재 상태/제약 파악 → 현실 인식
- 그 다음 결과물 정의 → 현실적인 범위
- 마지막 성공 기준 → 측정 가능한 종료점

---

## 실전 템플릿

### 기본형
```
"[목적]을 위해 [타겟]을 대상으로 [결과물]을 만들고 싶어.
지금은 [현재 상태]이고, [제약 조건]이 있어.
[성공 기준]이면 완성이야."
```

### 예시 1: 쇼츠 자동화
```
"수익화를 위해 20-30대 직장인을 대상으로 쇼츠 자동화 시스템을 만들고 싶어.
지금은 아이디어만 있고, 코딩 실력이 초보 수준이야.
일주일에 10개 쇼츠가 자동 생성되면 완성이야."
```

### 예시 2: 블로그
```
"애드센스 수익을 위해 IT 초보자를 대상으로 튜토리얼 블로그를 만들고 싶어.
지금은 워드프레스 설치까지 했고, 하루 1시간 투자 가능해.
월 방문자 1만 명 달성하면 1단계 완성이야."
```

### 예시 3: 업무 자동화
```
"시간 절약을 위해 나 자신을 대상으로 보고서 자동화 스크립트를 만들고 싶어.
지금은 수작업으로 하고 있고, Python 기초만 알아.
버튼 하나로 보고서가 생성되면 완성이야."
```

## 6요소 점검 체크리스트

### 1. 목적 점검
```
□ "왜?"에 한 문장으로 답할 수 있는가?
□ 목적이 2개 이상 섞여 있지 않은가?
□ 이 목적이 진짜 내 목적인가? (남의 조언 아닌가?)
```

### 2. 타겟 점검
```
□ 누가 이걸 쓰는지 명확한가?
□ 타겟이 "모든 사람"이 아닌가? (= 타겟 없음)
□ 타겟의 니즈/문제를 알고 있는가?
□ 타겟이 나 자신이라면, 그것도 명시했는가?
```

### 3. 현재 상태 점검
```
□ 지금까지 한 것을 나열할 수 있는가?
□ 막혀 있다면, 어디서 막혔는지 알고 있는가?
□ 이전 시도가 있다면, 왜 실패했는지 알고 있는가?
□ 진행률을 %로 말할 수 있는가?
```

### 4. 제약 점검
```
□ 시간 제한이 있는가? (하루 몇 시간, 언제까지)
□ 예산 제한이 있는가?
□ 기술적 제한이 있는가? (쓸 수 있는 도구)
□ 내 실력 수준을 정직하게 평가했는가?
```

### 5. 결과물 점검
```
□ 결과물을 구체적으로 설명할 수 있는가?
□ 결과물이 목적+타겟과 일치하는가?
□ 비슷한 결과물을 본 적 있는가? (레퍼런스)
□ 제약 조건 내에서 가능한 결과물인가?
```

### 6. 성공 기준 점검
```
□ 측정 가능한가? ("좋은" ❌ → "10개" ✅)
□ 달성 시점을 알 수 있는가?
□ 이 기준이 너무 높거나 낮지 않은가?
□ 결과물 완성 = 성공 기준 달성인가? (구분 명확)
```

## 흔한 실수

| 실수 | 문제 | 해결 |
|------|------|------|
| 목적 생략 | "뭔가 하고 싶어" | "왜?"를 3번 반복 |
| 타겟 = 모든 사람 | 아무도 아님 | "딱 한 명"으로 좁히기 |
| 현재 상태 과장 | "거의 다 됐어" | 실제 진행률 % 계산 |
| 제약 무시 | "일단 해보자" | 최악의 제약 먼저 확인 |
| 결과물 모호 | "좋은 거" | 레퍼런스 찾기 |
| 성공 기준 없음 | "되면 좋겠지" | 숫자로 정의 |
| 결과물 ↔ 성공 기준 혼동 | 구분 없이 섞임 | 결과물=무엇, 기준=수준 |

---

## 결과물 vs 성공 기준 구분

| 구분 | 정의 | 예시 |
|------|------|------|
| **결과물** | 무엇이 나오나 (형태) | 블로그, 스크립트, 앱 |
| **성공 기준** | 어느 수준이면 되나 (측정) | 월 1만 방문자, 10초 내 실행 |

**잘못된 예:**
- "좋은 블로그" → 결과물+기준 섞임

**올바른 예:**
- 결과물: "IT 튜토리얼 블로그"
- 성공 기준: "월 방문자 1만 명"
