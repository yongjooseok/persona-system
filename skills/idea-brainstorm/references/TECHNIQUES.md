# 브레인스토밍 기법 상세 가이드

## 1. SCAMPER

기존 아이디어/제품/서비스를 7가지 관점으로 변형한다.

| 단계 | 질문 | 예시 (커피숍) |
|------|------|---------------|
| **S**ubstitute (대체) | 무엇을 다른 것으로 바꿀 수 있나? | 커피 → 차, 바리스타 → 로봇 |
| **C**ombine (결합) | 무엇과 합칠 수 있나? | 커피숍 + 서점, 커피 + 알코올 |
| **A**dapt (적용) | 다른 분야에서 빌려올 건? | 드라이브스루 (패스트푸드에서) |
| **M**odify (변형) | 크기/형태/속성을 바꾸면? | 초소형 키오스크, 24시간 운영 |
| **P**ut to other use (용도 변경) | 다른 용도로 쓸 수 있나? | 낮엔 카페, 밤엔 바 |
| **E**liminate (제거) | 빼면 어떻게 될까? | 좌석 없는 테이크아웃 전용 |
| **R**everse (역발상) | 반대로 하면? | 고객이 직접 로스팅 |

**적용법:**
1. 주제 설정
2. 7가지 질문 순서대로 적용
3. 각 질문당 최소 2개 아이디어

## 2. 6 Thinking Hats

6가지 관점으로 순차적 사고. 모두가 같은 방향으로 생각.

| 모자 | 관점 | 질문 |
|------|------|------|
| 🤍 **하얀 모자** | 정보/사실 | 알고 있는 사실은? 필요한 데이터는? |
| ❤️ **빨간 모자** | 감정/직관 | 느낌은? 첫인상은? 좋아/싫어? |
| 🖤 **검은 모자** | 비판/위험 | 문제점은? 실패 가능성은? |
| 💛 **노란 모자** | 긍정/장점 | 장점은? 기회는? 최선의 시나리오는? |
| 💚 **녹색 모자** | 창의/대안 | 다른 방법은? 새로운 아이디어는? |
| 💙 **파란 모자** | 관리/정리 | 다음 단계는? 결론은? |

**적용법:**
1. 파란 모자로 시작 (목표 설정)
2. 하얀 → 빨간 → 검은 → 노란 → 녹색 순서
3. 파란 모자로 마무리 (정리)

## 3. How Might We (HMW)

문제를 기회 질문으로 전환한다.

**공식:** "어떻게 하면 [목표]를 달성하면서 [제약]을 극복할 수 있을까?"

**변환 예시:**

| 문제 | HMW 질문 |
|------|----------|
| 고객이 앱을 안 쓴다 | 어떻게 하면 고객이 매일 앱을 열고 싶게 만들까? |
| 비용이 너무 높다 | 어떻게 하면 품질 유지하며 비용을 절반으로 줄일까? |
| 경쟁이 심하다 | 어떻게 하면 경쟁 없는 새 시장을 만들까? |

**HMW 분해 기법:**
- 크게 만들기: "어떻게 하면 모든 사람이...?"
- 작게 만들기: "어떻게 하면 한 명만이라도...?"
- 반대로: "어떻게 하면 오히려 [반대 상황]을...?"
- 가정 제거: "만약 [제약]이 없다면...?"

## 4. Random Stimulus (무작위 자극)

막혔을 때 랜덤 단어/이미지로 연상 유도.

**방법:**
1. 랜덤 단어 선택 (사전, 뉴스 헤드라인, 주변 사물)
2. 단어의 속성 나열 (특징, 용도, 연관어)
3. 각 속성을 주제와 강제 연결

**예시: "고객 유치 방법" + 랜덤 단어 "나무"**

| 나무의 속성 | 강제 연결 |
|-------------|-----------|
| 뿌리 | 입소문 네트워크 → 기존 고객의 추천 프로그램 |
| 성장 | 시간에 따른 혜택 증가 → 가입 기간별 등급제 |
| 그늘 | 보호/쉼터 → 고객 전용 커뮤니티 |
| 계절 변화 | 시즌별 이벤트, 계절 한정 상품 |

## 5. Mind Mapping

중심 주제에서 방사형 확장.

**구조:**
```
                    [가지 1-1]
                   /
            [가지 1] ── [가지 1-2]
           /
[중심 주제] ── [가지 2] ── [가지 2-1]
           \
            [가지 3] ── [가지 3-1]
                   \
                    [가지 3-2]
```

**규칙:**
1. 중심에 핵심 주제 (1-3단어)
2. 1차 가지: 큰 카테고리 (3-7개)
3. 2차 가지: 세부 아이디어
4. 색상/아이콘으로 구분
5. 연결선으로 관계 표시

**텍스트 마인드맵 예시:**
```
[AI 에이전시 서비스]
├─ 자동화
│  ├─ 이메일 자동 응답
│  ├─ 리포트 자동 생성
│  └─ 데이터 수집 봇
├─ 콘텐츠
│  ├─ 블로그 작성
│  ├─ SNS 포스팅
│  └─ 영상 스크립트
└─ 컨설팅
   ├─ AI 도입 진단
   ├─ 워크플로우 설계
   └─ 교육/온보딩
```
