# idea-brainstorm

> 아이디어 브레인스토밍 스킬

## 개요

| 항목      | 내용                       |
| --------- | -------------------------- |
| 소속      | 기획부 (Planning Division) |
| 유형      | 발상 스킬                  |
| 산출물    | 아이디어 목록 (Markdown)   |
| 저장 위치 | `outputs/ideas/`           |

## 입력 (Input)

| 필수 | 항목      | 설명                                           |
| :--: | --------- | ---------------------------------------------- |
|  O   | 주제/문제 | 브레인스토밍할 주제 또는 해결할 문제           |
|  -   | 맥락      | 배경 상황, 제약 조건                           |
|  -   | 방향성    | 원하는 아이디어 방향 (혁신적/실용적/저비용 등) |
|  -   | 목표 개수 | 도출할 아이디어 수 (기본: 10개)                |
|  -   | 모드      | 자유발산 / 기법지정 / 평가포함 (기본: 자유발산) |

## 사용 모드

| 모드 | 트리거 | 동작 |
|------|--------|------|
| 1. 자유 발산 | "아이디어 내줘", "브레인스토밍 해줘" | 맥락 파악 → 기법 자동 선택 → 아이디어 10개+ |
| 2. 기법 지정 | "SCAMPER로 분석해줘", "6 Hats 적용해줘" | 해당 기법 심층 적용 |
| 3. 평가 포함 | "평가까지 해줘", "실행 가능한 것 추려줘" | 발산 + 매트릭스 평가 + Top 3 선정 |

## 브레인스토밍 기법

### 상황별 추천

| 상황 | 추천 기법 | 특징 |
|------|-----------|------|
| 기존 것 개선 | SCAMPER | 7가지 변형 질문으로 체계적 확장 |
| 다각도 검토 | 6 Thinking Hats | 6가지 관점으로 균형잡힌 발상 |
| 문제 재정의 | How Might We | 문제를 기회로 전환 |
| 막힘 돌파 | Random Stimulus | 무작위 자극으로 연상 유도 |
| 구조 시각화 | Mind Mapping | 중심에서 방사형 확장 |

**기법 상세:** [references/TECHNIQUES.md](references/TECHNIQUES.md)

## 실행 단계 (SOP)

### 1단계: 맥락 파악

- [ ] 주제/문제 명확히 이해
- [ ] 맥락과 제약 조건 파악
- [ ] 성공 기준 확인: 어떤 아이디어면 좋은 건가?

### 2단계: 기법 선택

- [ ] 상황에 맞는 기법 선택 (상황별 추천 표 참고)
- [ ] 모드 확인 (자유발산/기법지정/평가포함)

### 3단계: 확산 (Diverge)

**핵심 원칙:** 판단 유보 / 양 우선 / 연결 장려 / 엉뚱함 환영

- [ ] 선택한 기법으로 아이디어 10-15개 생성
- [ ] 각 아이디어에서 파생 아이디어 추가
- [ ] 역발상, 유추법 시도

### 4단계: 조합

- [ ] 유사 아이디어 그룹화
- [ ] 서로 다른 아이디어 조합 시도

### 5단계: 수렴 (Converge)

- [ ] 실현 가능성 기준 필터링
- [ ] 각 아이디어 장단점 정리
- [ ] 실행 난이도 평가 (상/중/하)

### 6단계: 정리

- [ ] 최종 아이디어 목록 작성
- [ ] 우선순위 제안 (Top 3)
- [ ] 다음 단계 액션 제안

## 출력 형식

### 기본 (모드 1, 2)

```markdown
---
title: [주제] 아이디어
date: YYYY-MM-DD
technique: [적용 기법]
status: draft
---

## 요약

- 총 개수: N개
- 적용 기법: [기법명]

## 아이디어 목록

### 1. [아이디어명]

- **설명**: ...
- **장점**: ...
- **단점**: ...

### 2. [아이디어명]
...

## 조합 아이디어

- [아이디어 A] + [아이디어 B] → [새로운 가능성]

## 다음 단계

1. ...
```

### 평가 포함 (모드 3)

위 내용에 추가:

```markdown
## 평가 매트릭스

| 아이디어 | 실현가능성 | 임팩트 | 난이도 | 우선순위 |
|----------|------------|--------|--------|----------|
| ... | ⭐⭐⭐ | ⭐⭐ | 중 | 1 |

## Top 3 추천

1. **[아이디어명]** — [추천 이유]
2. ...

## 즉시 실행 가능한 액션

- [ ] ...
```

## 품질 체크리스트

### 필수 항목

- [ ] 주제와 관련 있는 아이디어인가?
- [ ] 최소 10개 이상의 아이디어가 있는가?
- [ ] 각 아이디어가 구체적으로 설명되어 있는가?

### 권장 항목

- [ ] 다양한 관점의 아이디어가 포함되어 있는가?
- [ ] 조합 아이디어가 시도되었는가?
- [ ] 우선순위 근거가 명확한가?

## 하지 않는 것

- 아이디어 선택/결정 (기획부장 또는 CEO의 역할)
- 실행 계획 수립 (별도 스킬)
- 품질 판정 (검수부의 역할)

## 호출 예시

```
"idea-brainstorm 스킬로 브레인스토밍 해줘"

- 주제: 블로그 구독자 늘리기
- 맥락: 기술 블로그, 월 방문자 1000명
- 방향성: 저비용, 지속 가능한 방법
- 모드: 평가포함
```

## 핵심 원칙

- **많이 > 좋게**: 처음엔 양으로 승부
- **조합의 힘**: 평범한 것도 결합하면 새로워짐
- **제약은 친구**: 제약이 명확할수록 창의성 발휘
