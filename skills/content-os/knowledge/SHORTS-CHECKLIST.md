# 쇼츠 스크립트 검수 체크리스트 (지식파일)

## 사용 방법

1. 스크립트 검수 시 아래 항목 순서대로 체크
2. 모든 ★필수 항목 통과 → **통과**
3. ★필수 항목 1-2개 미통과 → **수정 필요**
4. ★필수 항목 3개+ 미통과 → **재작성**

---

## 1. 구조 검수 (Structure)

### ★ 필수 항목

| 항목 | 기준 | 체크 |
|------|------|------|
| 글자수 | 100-130자 | □ |
| 줄 수 | 12-14줄 | □ |
| 훅 | 1줄, 호기심 유발 | □ |
| 마무리 | 반전/위트 있음 | □ |

### 권장 항목

| 항목 | 기준 | 체크 |
|------|------|------|
| 한 줄 길이 | 5-18자 (가변) | □ |
| 섹션 비율 | 10%-80%-10% | □ |

## 3. 언어 검수 (Language)

### ★ 필수 항목

| 항목 | 기준 | 체크 |
|------|------|------|
| 필수 패턴 | 2개 이상 사용 | □ |
| 금지어 | 미사용 | □ |

### 필수 패턴 목록
```
□ "~라는", "~한다는데", "~라고 함"
□ "그러나", "그런데", "의외로"
□ "결국", "심지어", "특히"
```

### 금지어 목록
```
❌ "~입니다", "~합니다"
❌ "매우", "정말로", "굉장히"
❌ "최고의", "완벽한"
```

아래 중 하나 적용됐는지:

| 공식 | 체크 |
|------|------|

## 판정 기준

### ✅ 통과 (상)
```
- 모든 ★필수 항목 통과
- 수정 없이 사용 가능
```

### ⚠️ 수정 필요 (중)
```
- ★필수 항목 1-2개 미통과
- 간단한 수정으로 해결 가능
```

### ❌ 재작성 (하)
```
- ★필수 항목 3개 이상 미통과
- 구조적 문제 있음
```

---

## 빠른 검수 (5초 체크)

시간 없을 때 최소한 이것만:

```
1. □ 글자수 80-120자?
2. □ 첫 줄 호기심 있나?
3. □ 끝 줄 반전/위트 있나?
4. □ 반말 사용했나?
5. □ 정보 + 감정 둘 다 있나?
```

→ 5개 모두 ✅ = 통과
→ 1-2개 ❌ = 수정
→ 3개+ ❌ = 재작성

---

## 피드백 작성 원칙

1. **구체적으로**: "훅 약함" ❌ → "첫 줄에 호기심 없음, '~한다는' 추가" ✅
2. **수정안 제시**: 문제만 지적 ❌ → 대안 함께 ✅
3. **우선순위**: ★필수 항목 먼저
4. **긍정 먼저**: 잘된 점 언급 후 수정사항
