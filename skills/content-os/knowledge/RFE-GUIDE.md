# RFE 추출 가이드 (지식파일)

## RFE란?

**Reference for Execution** - 실행규칙 레벨
- "어떻게 구현할 것인가?"
- 실행을 위한 구체적 규칙 세트

---

## 추출 순서

> ⚠️ **RFE를 REF보다 먼저 추출** (실행규칙 → 전략)

```
샘플 읽기 → 요소 해체 → 패턴 발견 → 규칙화 → RFE 문서화
```

---

## 1. Structure Rules 추출

### 측정 항목
```
□ 전체 길이 (시간/글자수)
□ 섹션 구성 (시작/중간/끝)
□ 각 섹션 비율 (%)
□ 줄 수 / 문장 수
□ 한 줄당 글자 수
```

### 추출 방법
1. 샘플별 길이 측정 (숫자로)
2. 섹션 경계 파악
3. 평균값 + 범위 도출

### 출력 형식
```markdown
## Structure Rules

### 전체 스펙
| 항목 | 규칙 |
|------|------|
| 길이 | [n-m초] |
| 글자수 | [n-m자] |
| 줄 수 | [n-m줄] |

### 섹션 구성
| 섹션 | 비율 | 글자수 |
|------|------|--------|
| 시작 | [n]% | [n]자 |
| 중간 | [n]% | [n]자 |
| 끝 | [n]% | [n]자 |
```

## 2. Tone & Manner 추출

### 측정 항목
```
□ 격식 수준 (반말/존댓말/혼용 비율)
□ 감정 온도 (냉정/중립/따뜻)
□ 전문성 수준 (전문/일반/캐주얼)
□ 이모티콘/특수문자 사용 여부
```

### 추출 방법
1. 샘플에서 말투 패턴 수집
2. 감정 표현 빈도 체크
3. 공통 톤 정의

### 출력 형식
```markdown
## Tone & Manner

### 격식
| 항목 | 규칙 |
|------|------|
| 말투 | [반말 n% / 존댓말 n%] |
| 느낌 | [한 단어] |

### 감정 표현
| 감정 | 표현 | 사용 위치 |
|------|------|----------|
| [감정] | [표현] | [위치] |
```

---

## 3. Language Rules 추출

### 측정 항목
```
□ 자주 쓰는 단어/표현 (빈도순)
□ 문장 시작 패턴
□ 연결어 사용
□ 금지어/피해야 할 표현
□ 숫자/단위 표기법
```

### 추출 방법
1. 샘플에서 반복 패턴 추출
2. 빈도 높은 표현 리스트업
3. 사용 안 하는 표현도 기록

### 출력 형식
```markdown
## Language Rules

### 필수 패턴 (최소 n개 사용)
```
□ [패턴 1]
□ [패턴 2]
```

### 금지어
```
❌ [금지 표현 1] - [이유]
❌ [금지 표현 2] - [이유]
```

### 숫자 표현
```
✅ [올바른 예]
❌ [잘못된 예]
```
```

## 4. Format Rules 추출

### 측정 항목
```
□ 줄바꿈 규칙 (몇 문장마다)
□ 강조 방법 (볼드/위치)
□ 리스트 사용 여부
□ 특수 기호 사용
```

### 추출 방법
1. 샘플의 시각적 구조 분석
2. 줄바꿈 위치 패턴 파악
3. 강조 표현 방식 정리

### 출력 형식
```markdown
## Format Rules

### 줄바꿈 규칙
```
✅ [규칙 1]
✅ [규칙 2]
```

### 강조 규칙
```
✅ [허용]
❌ [금지]
```
```

## 5. Filter 추출

### 측정 항목
```
□ 반드시 있어야 하는 것 (통과 조건)
□ 있으면 안 되는 것 (차단 조건)
□ 최소/최대 기준
```

### 추출 방법
1. 성공 샘플의 공통 필수 요소 추출
2. 실패 샘플의 공통 문제점 추출 (있다면)
3. 체크리스트화

### 출력 형식
```markdown
## Filter

### ✅ 통과 조건 (전부 만족)
```
□ [조건 1]
□ [조건 2]
```

### ❌ 차단 조건 (하나라도 해당 시 수정)
```
□ [조건 1]
□ [조건 2]
```
```

## 6. 예시 섹션 작성

### 필수 포함
- Good Example (통과 예시)
- Bad Example (실패 예시)
- 각각 왜 좋은지/나쁜지 설명

### 출력 형식
```markdown
## 예시

### Good Example ✅
```
[예시 내용]
```
- [좋은 이유 1]
- [좋은 이유 2]

### Bad Example ❌
```
[예시 내용]
```
- 문제: [문제점]
```

## 품질 체크

RFE 완성 후 확인:
```
□ 이 RFE만 보고 콘텐츠를 만들 수 있는가?
□ 통과/차단 기준이 명확한가?
□ 주관적 판단 없이 체크 가능한가?
□ 모든 규칙이 숫자로 측정 가능한가?
□ 예시가 규칙을 잘 보여주는가?
```

---

## 주의사항

1. **측정 가능하게**: "좋은 훅" ❌ → "15자 이내 질문형" ✅
2. **예외 최소화**: 규칙은 단순할수록 좋음
3. **Good/Bad 필수**: 예시 없는 규칙은 불완전
