# 콘텐츠 자동화 시스템 지침

## 핵심 원칙

1. **기준 먼저, 도구 나중**: 샘플 분석 → REF/RFE 추출 → 콘텐츠 생성
2. **REF는 "왜"**, **RFE는 "어떻게"**: 분리해서 관리
3. **측정 가능하게**: "좋은" ❌ → "15자 이내" ✅

---

## 역기획 분석 시

### 순서
```
샘플 수집 → RFE 추출 → REF 추출 → 프로필 저장
```

### RFE 추출 항목
- Structure (구조): 길이, 섹션, 비율
- Tone (톤): 격식, 감정표현
- Language (언어): 필수패턴, 금지어
- Format (포맷): 줄바꿈, 강조
- Filter (필터): 통과조건, 차단조건

### REF 추출 항목
- Value Proposition: 타겟, 가치, 차별점
- Psychological Flow: 감정 곡선
- Strategic Intent: 목표 행동, 핵심 메시지

### 참조 파일
- RFE 추출: `REF-GUIDE.md` 참조
- REF 추출: `RFE-GUIDE.md` 참조

## 쇼츠 스크립트 생성 시

### v1 스펙 (18-20초)
- 글자수: 100-130자
- 줄 수: 12-14줄
- 한 줄: 5-18자 (가변)
- 구성: 훅(1줄) + 본문(10-12줄) + 마무리(1줄)

### 필수 체크
- [ ] 글자수 100-130자
- [ ] 첫 줄 호기심 유발
- [ ] 끝 줄 반전/위트
- [ ] 반말 사용
- [ ] 정보 + 감정 둘 다

### 훅 공식 (첫 줄)
- "[명사]가 [동사]한다는"
- "[대상]이 [상황]이라는"
- "사람들이 [행동]하는 이유"

### 마무리 공식 (끝 줄)
- 질문형: "~인가?", "~인데?"
- 위트형: "~ㅋㅋ", "~ㄷㄷ"

### 참조 파일
- 상세 규칙: `SHORTS-V1-PROFILE.md` 참조
- 검수: `SHORTS-CHECKLIST.md` 참조

## 검수 시

### 판정 기준
- **통과**: 모든 필수 항목 통과
- **수정 필요**: 필수 1-2개 미통과
- **재작성**: 필수 3개+ 미통과

### 피드백 원칙
1. 문제점 + 수정안 함께 제시
2. 잘된 점 먼저 언급
3. 우선순위대로 나열

---

## 버전 관리

### 승급 조건 (v1 → v2)
- 10개 이상 제작 완료
- 성공 패턴 3개 이상 확인
- RFE 안정화

### 피드백 루프
```
제작 → 발행 → 성과 측정 → 프로필 업데이트 → 반복
```
