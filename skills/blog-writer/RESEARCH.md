# 주제 리서치 가이드

## 리서치 목표

1. **검색 수요 확인**: 사람들이 실제로 검색하는 주제인가?
2. **경쟁 분석**: 기존 콘텐츠의 강점/약점은?
3. **차별화**: 우리만의 가치는 무엇인가?

---

## Step 1: 키워드 리서치

### 핵심 키워드 도출

```
메인 키워드: [주제의 핵심 단어]
예: "Python 설치", "Git 사용법", "Docker 입문"
```

### 관련 키워드 확장

WebSearch를 활용하여 다음을 검색:

1. "[메인 키워드] 방법"
2. "[메인 키워드] 초보"
3. "[메인 키워드] 쉽게"
4. "[메인 키워드] 오류"

### 롱테일 키워드 수집

초보자가 실제로 검색할 구체적인 질문들:

- "~하는 방법"
- "~란 무엇인가"
- "~ 시작하기"
- "~ 설치 오류 해결"

---

## Step 2: 경쟁 콘텐츠 분석

### 상위 콘텐츠 조사

검색 결과 상위 5개 글을 분석:

| 순위 | 제목 | 강점 | 약점 | 차별화 기회 |
|------|------|------|------|-------------|
| 1 | | | | |
| 2 | | | | |
| 3 | | | | |
| 4 | | | | |
| 5 | | | | |

### 분석 포인트

**강점 파악:**
- 설명이 명확한가?
- 시각 자료가 풍부한가?
- 실습 예제가 있는가?

**약점 파악:**
- 너무 어려운 설명?
- 오래된 정보?
- 실습 불가능?

---

## Step 3: 초보자 관점 질문 수집

### 필수 수집 질문

1. **기본 개념**: "이게 뭔가요?"
2. **필요성**: "왜 배워야 하나요?"
3. **시작점**: "어디서부터 시작하나요?"
4. **흔한 오류**: "왜 안 되나요?"
5. **다음 단계**: "이 다음엔 뭘 배우나요?"

### 질문 수집 방법

- Stack Overflow 관련 질문
- Reddit/커뮤니티 질문
- YouTube 댓글
- 블로그 댓글

---

## Step 4: 차별화 포인트 정의

### Quiet Notes의 차별화 전략

| 기존 콘텐츠 문제점 | Quiet Notes 해결책 |
|-------------------|-------------------|
| 영어 문서 위주 | 한국어로 쉽게 설명 |
| 전문 용어 남발 | 모든 용어 풀어서 설명 |
| 설치만 다룸 | 설치 + 첫 실행 + 간단한 예제 |
| 텍스트만 있음 | 스크린샷 + 코드 + 결과 화면 |

### 우리 글의 핵심 가치

```
"처음 접하는 사람도 15분 안에 따라할 수 있는 가이드"
```

## 리서치 결과 템플릿

리서치 완료 후 다음 형식으로 정리:

```markdown
## 리서치 요약

**주제**: [주제명]
**메인 키워드**: [키워드]
**타겟 독자**: [구체적 대상]

### 핵심 인사이트

1. 초보자가 가장 궁금해하는 것:
2. 기존 콘텐츠의 주요 문제점:
3. 우리의 차별화 포인트:

### 반드시 다뤄야 할 내용

- [ ]
- [ ]
- [ ]

### 참고 자료

- [URL 1]
- [URL 2]
```

- 공식 문서와 교차 검증
- 실제로 따라해보며 검증
