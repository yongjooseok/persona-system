# Quiet Notes 블로그 글 작성 스킬

## 개요

Quiet Notes는 AI/IT 튜토리얼 블로그로, 개발 초보자를 타겟으로 합니다.
이 스킬은 고품질 블로그 글 작성의 전체 과정을 안내합니다.

## 워크플로우

```
1. 리서치 → 2. 제목 생성 → 3. 글 작성 → 4. 검수
```

### Phase 1: 주제 리서치

상세 가이드: [RESEARCH.md](RESEARCH.md)

**핵심 체크포인트:**
- 검색 트렌드 확인 (Google Trends, 관련 키워드)
- 경쟁 콘텐츠 분석 (상위 5개 글)
- 초보자가 궁금해할 질문 수집
- 차별화 포인트 도출

### Phase 2: 클릭률 높은 제목 생성

상세 가이드: [TITLES.md](TITLES.md)

**제목 공식:**
1. 숫자 + 키워드 + 결과: "Python 5분 만에 설치하는 3가지 방법"
2. How-to 형식: "초보자도 쉽게 따라하는 Git 사용법"
3. 문제 해결형: "Docker 설치 오류? 이렇게 해결하세요"

### Phase 3: 초보자 친화적 글 작성

상세 가이드: [WRITING.md](WRITING.md)

**필수 원칙:**
- 전문 용어는 반드시 설명
- 단계별 스크린샷/코드 포함
- 복사 가능한 코드 블록
- 예상 결과 미리 보여주기

### Phase 4: 검수 체크리스트

상세 가이드: [CHECKLIST.md](CHECKLIST.md)

**최종 검토 항목:**
- [ ] 제목이 내용을 정확히 반영하는가?
- [ ] 초보자가 따라할 수 있는가?
- [ ] 코드가 실제로 동작하는가?
- [ ] 오타/문법 오류가 없는가?

## 빠른 시작

사용자가 "블로그 글 써줘" 또는 "튜토리얼 작성해줘"라고 하면:

1. 주제 확인
2. 리서치 수행 (RESEARCH.md 참조)
3. 제목 후보 3개 제시 (TITLES.md 참조)
4. 선택된 제목으로 글 작성 (WRITING.md 참조)
5. 검수 체크리스트 실행 (CHECKLIST.md 참조)

## 출력 형식

최종 결과물은 다음 형식으로 제공:

```markdown
# [제목]

## 들어가며
(왜 이 글을 읽어야 하는지)

## 목차
(자동 생성)

## 본문
(단계별 설명)

## 마무리
(요약 + 다음 단계 안내)
```
