# 전문가 콘텐츠 전략가 — 페르소나 깊이 구체화

> 메타포(전문가) × 직원(콘텐츠 전략가) × 깊이(5층위)를 한 번에 묶어 실제 정의 파일 v0 골격까지.

---

## 1. 출발점 — 전문가 메타포 위에서 콘텐츠 전략가란

```
의뢰인: 로그 (콘텐츠 전략 영역의 비전문가는 아니지만, 외부 시각이 필요)
   ↓ 의뢰
전문가: 콘텐츠 전략가 (콘텐츠의 본질·구조·표현에 대한 외부 권위)
   ↓ 진단·제안·초안
의뢰인: 로그가 검토·결정·최종 편집
   ↓
산출물: 로그의 이름으로 세상에 나간다
```

**핵심 함의:**
- 전략가는 **조언·진단·초안 제출**이 책임. 결정·창작 권위는 로그.
- 그래서 전략가의 산출물은 "완성품"이 아니라 **"의뢰인이 선택·결정할 수 있는 옵션·근거·초안"**.
- 페르소나는 "내가 더 잘 안다"가 아니라 "이런 시각도 있다"의 톤.

---

## 2. 페르소나 정의 6요소

CrewAI식 4요소(Role / Goal / Backstory / Tools)에 **깊이 2요소(Lens / Algorithm)**를 추가.

| 요소 | 정의 | 깊이 |
|---|---|---|
| **Role** | 직책·전문 영역 | 표층 |
| **Goal** | 이 전문가가 추구하는 것 | 중간 |
| **Backstory** | 경력·경험·신념의 형성 배경 | 중간 |
| **Tools** | 보유 스킬·도구 | 표층 |
| **Lens** | 사물·콘텐츠를 보는 관점 | 심층 |
| **Algorithm** | 문제 받았을 때 사고의 순서 | 심층 |

뒤의 둘(Lens, Algorithm)이 페르소나 깊이의 핵심. 앞의 넷만 있으면 "이력서"고, 뒤의 둘이 있어야 "사람"이 된다.

---

## 3. 페르소나 깊이 5층위 — 콘텐츠 전략가에 적용

이전 차원 분석의 5층위에 콘텐츠 전략가를 입혀보면:

### 층위 1 — 표층: 말투·톤
- 의뢰인을 어떻게 부르는가 ("대표님" / "로그 님" / 이름)
- 어조: 단정형 vs 완곡형, 직접 vs 우회
- 문장 길이의 평균값
- 감정의 진폭 (담담 vs 격정)

**예시 (박웅현 풍 가정):**
> "이 문장에서 본질이 보이지 않습니다. 한 번 더 들어가 보시죠."
**예시 (정철 풍 가정):**
> "사람이 안 보여요. 사람을 보여주세요."

### 층위 2 — 어휘·비유 패턴
- 어떤 어휘를 끌어오는가 (인문학 / 광고 / 마케팅 데이터 / 일상)
- 비유의 출처 (자연·일상·예술·통계·기술)
- **클리셰 회피 패턴** — 어떤 표현을 피하는가
- 고유 표현 — 이 전략가만 자주 쓰는 표현

### 층위 3 — 판단 기준
콘텐츠 전략가가 "좋다/나쁘다"를 가르는 기준:

| 후보 기준 | 우선순위 (예시) |
|---|---|
| 본질 (메시지의 깊이) | 1 |
| 발견 (새 시각) | 2 |
| 공감 (사람에 닿는가) | 3 |
| 정확성 | 4 |
| 조회수·CTR | 5 |
| 새로움·트렌드 | 6 |

순위가 다른 두 전략가는 같은 의뢰에 다른 답을 낸다. **이 순위 자체가 페르소나의 정체성.**

### 층위 4 — 사고 알고리즘
콘텐츠 의뢰를 받았을 때 어떤 순서로 생각하는가.

**예시 — "본질 우선형" 알고리즘:**
```
1. "이 콘텐츠가 진짜 말하려는 한 문장은?" 추출
2. 그 문장의 가장 진부한 표현부터 나열
3. 진부함의 반대편을 찾는다
4. 본질을 새로 보는 비유 1개 발굴
5. 비유 ↔ 본질을 잇는 도입·전개·끝맺음
6. 의뢰인에게 "왜 이 비유인가" 설명 가능한 형태로 정리
```

**예시 — "행동 유발형" 알고리즘 (대비):**
```
1. 독자가 이 글 읽고 무엇을 하길 바라는가?
2. 그 행동을 막는 심리적 장벽 3개 식별
3. 장벽 별로 해소 단계 설계
4. 클릭 → 스크롤 → 결정의 각 지점 메시지 배치
5. CTA의 위치·문구 최적화
```

같은 콘텐츠 전략가라는 직책이지만 알고리즘이 다르면 **완전히 다른 직원**이 된다. 이게 페르소나 깊이의 본질.

### 층위 5 — 세계관
왜 그렇게 하는가의 근본.

**예시 (본질 우선형):**
> "사람은 진짜를 본 적이 있으면 가짜를 못 견딘다. 콘텐츠의 임무는 진짜를 한 번 보여주는 것이다."

**예시 (행동 유발형):**
> "메시지는 행동으로 이어지지 않으면 의미 없다. 좋은 콘텐츠는 사람을 움직이게 한다."

세계관이 다르면 같은 클라이언트 같은 의뢰에서도 산출물이 갈린다.

## 4. 페르소나 정의 파일 v0 골격

```markdown
---
type: persona
domain: content
metaphor: expert
version: 0.1
---

# [페르소나 이름]
콘텐츠 전략가 (의뢰형 외부 전문가)

## Role
- 직책: 콘텐츠 전략가
- 전문 영역: 브랜드 메시지·콘텐츠 구조·표현 전략
- 의뢰 형태: 진단 / 전략 제안 / 초안 작성 / 검수

## Goal
[이 전략가가 모든 작업에서 추구하는 것 1문장]
예시: "의뢰인의 콘텐츠가 본질에 닿게 한다."

## Backstory
[경력·경험·신념의 형성. 3~5문단.]
- 어디서 일했는가
- 어떤 사건을 통해 지금의 신념을 얻었는가
- 어떤 콘텐츠를 만들어 왔는가
- 왜 이 일을 하는가

## Lens (관점)
- 콘텐츠를 볼 때 우선 보는 것:
  1. ___
  2. ___
  3. ___
- 무시하는 것 / 후순위:
  - ___

## Algorithm (사고 순서)
의뢰 받으면 다음 순서로 사고:
1. ___
2. ___
3. ___
4. ___
5. ___

## Voice (말투·톤)
- 의뢰인 호칭: ___
- 어조: ___
- 문장 길이 경향: ___
- 자주 쓰는 표현: ___
- 피하는 표현: ___

## Tools (보유 스킬)
- @skills/content-os
- @skills/idea-brainstorm
- @skills/blog-writer
- @skills/question-design

## Output Standards (담당 산출물)
- @output-standards/content-diagnosis-report.md
- @output-standards/content-strategy-brief.md
- @output-standards/blog-draft.md

## Boundaries (위임 한계)
이 전략가가 **결정하지 않는 것:**
- 최종 게재 여부 → 로그
- 브랜드 톤의 변경 → 로그
- 새 제품 라인 결정 → 로그
이 전략가가 **하지 않는 것:**
- 디자인 / 영상 제작
- 광고 매체 집행
- 데이터 분석 (별도 직원)

## Worldview (세계관)
[왜 이 일을 이렇게 하는지의 근본 1~2문장]
```

## 5. 페르소나 깊이 검증 — 3가지 테스트

만들고 나면 깊이가 충분한지 검증해야 한다.

**테스트 ① — 페르소나 대비 테스트**
- 같은 의뢰를 페르소나 A(본질 우선형)와 페르소나 B(행동 유발형)에게 준다
- 출력이 명확히 다른가? 같다면 깊이 부족 (Algorithm 층위까지 못 내려간 것)

**테스트 ② — 도메인 이동 테스트**
- 같은 페르소나에게 익숙한 의뢰(블로그)와 낯선 의뢰(제품 패키지 카피)를 준다
- 둘 다 일관된 사고 패턴(Algorithm)이 보이는가? 표면만 흉내내면 도메인 바뀔 때 깨진다

**테스트 ③ — 자기 설명 테스트**
- 페르소나에게 "왜 그렇게 결정했는지" 물어본다
- Worldview·Lens·Algorithm으로 답할 수 있나? "그냥요"라고 하면 깊이 부족

---

## 6. 페르소나 원형 — 결정 필요사항

가장 큰 결정: **원형이 누구/무엇인가.**

**옵션 A — 실제 인물 차용**
- 박웅현, 정철, 광고계 인물, 카피라이터 등
- 장점: 풍부한 참조 자료 (책·인터뷰·작업물)
- 단점: 인물의 한계·이미지를 그대로 가져옴. 그 인물의 약점도 따라옴

**옵션 B — 가상 인물 구축**
- 로그가 원하는 자질만 골라 조합
- 장점: 제약 없음, 원하는 인물 설계 가능
- 단점: Backstory 구체성 떨어짐, "진짜 같음" 부족

**옵션 C — 하이브리드**
- 실제 인물 2~3명의 자질을 합성 + 가상 Backstory
- 장점: 풍부함 + 자유도
- 단점: 정체성 흐려질 수 있음

**로그의 단서:**
- Creamhaus 작업에서 박웅현/정철 스타일 카피 경험 있음
- 둘은 사고 방식이 다름 (박웅현: 본질·발견 / 정철: 사람·진정성)
- 둘을 합치면 흥미롭지만, 충돌하는 자질도 있음

**개인적 추론(추정):** 옵션 C(하이브리드)가 가장 유용하지만, 1차 시범에는 **옵션 A(실제 인물 1명)** 가 더 빨리 검증 가능. 작동 보면 후속 직원을 다른 원형으로 만들기 쉬움.

## 7. 1차 시범 제안

지금까지 결정·합의된 것을 합치면:

```
[1차 시범 직원: 콘텐츠 전략가 v0.1]
- 메타포: 전문가 (의뢰형)
- 원형: 실제 인물 1명 (박웅현 or 정철 or 다른 사람 — 로그 결정)
- 운영 방식: MD 정의 파일
- 깊이: 6요소 + 5층위 모두 채움
- 첫 산출물 표준: ___ (다음 결정)
- 검증: 3가지 테스트 통과
```

**1차 시범에서 의도적으로 안 하는 것:**
- 팀 협업 (단일 직원)
- Sub-agent 인프라
- 메모리·학습
- 디렉터·오케스트레이터

이걸 다 안 하기로 한 이유: **페르소나 깊이 공식 자체가 검증되지 않은 상태에서 협업·인프라·학습을 얹으면 무엇이 작동하고 무엇이 안 되는지 분리가 안 됨.**

## 8. 불확실 지점

- 6요소(Role/Goal/Backstory/Tools/Lens/Algorithm) 구조는 CrewAI + 자체 보강. **학술 표준 아님.**
- 5층위 분류는 일반론에 가까우나 정확한 출처는 명시 어려움. 인지언어학·HCI 분야의 분산된 논의.
- "원형 옵션 C가 후속에 유리"라는 권고는 합리적 추론이지 검증된 답 아님.
- 실제 인물(박웅현·정철)의 사고 알고리즘은 그들의 책·인터뷰에서 재구성 가능하지만, 100% 정확하진 않음 (사후 재구성의 한계).
